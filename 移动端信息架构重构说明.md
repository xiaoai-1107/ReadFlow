# 移动端信息架构重构说明

## 1. 本轮调整目标

本轮移动端重构的目标不是局部补样式，而是把移动端阅读页从“阅读管理页”收口为“以阅读为中心的移动阅读器”。

调整原则只有一条：

`正文优先，功能按需进入，状态轻量贴边。`

## 2. 为什么这样调整

上一轮移动端稿的主要问题不在视觉风格，而在信息架构：

1. 顶部区域偏重，首屏被导航和模式控件占走太多高度。
2. `Structure / My Study / Details` 作为大按钮直接插在正文前，阅读一开始就被入口打断。
3. `Structure` 的存在方式更像页面内容，而不是按需打开的导航工具。
4. 恢复阅读提示和底部状态区都偏厚，进一步压缩了正文面积。

这会让用户第一眼感受到的是“我要操作哪些功能”，而不是“我正在读什么”。

因此本轮调整把移动端重新定义为：

1. 顶部只保留最轻的导航与模式切换。
2. 正文在首屏尽快出现。
3. 次级功能入口移出正文主流。
4. 当前段落详情、目录、My Study 都改为按需打开。

## 3. 和 PC 的对应关系

本轮没有重新定义移动端产品逻辑，只调整了移动端的承载方式。

双端统一关系如下：

### Structure

- PC：左侧结构导航
- Mobile：通过 `Structure` 打开态的 sheet 进入

产品语义不变：
- 都是文档结构导航
- 都服务跳转与定位
- 都不进入正文主流

### My Study

- PC：左侧 `My Study`
- Mobile：通过二级入口打开 `My Study` sheet

产品语义不变：
- 都是“我的沉淀入口”
- 都用于进入 `Highlights / Tags / Recently Read`
- 都不是当前段落详情区

### Details

- PC：右侧详情面板
- Mobile：底部详情弹层

产品语义不变：
- 都围绕当前段落
- 都承载段落位置、翻译状态、高亮摘要、标签摘要

### 正文

- PC：多区域中的主阅读流
- Mobile：单列连续正文阅读流

产品语义不变：
- 原文模式只显示英文正文
- 译文模式是一段原文 + 一段译文

## 4. 哪些元素移出了首屏

为保证阅读优先，本轮有意识把以下元素移出了首屏正文前区域：

1. `Structure`
2. `My Study`
3. `Details`
4. 厚重的恢复阅读提示
5. 厚重的底部状态条

处理方式如下：

1. `Structure` 改为按需打开的 sheet
2. `My Study` 改为二级入口打开
3. `Details` 改为底部弹层
4. 恢复阅读提示改为更轻的顶部提示
5. 底部状态区压缩为更薄的全局轻状态层

## 5. 如何保证阅读优先

本轮通过 4 个结构动作来保证“阅读优先”：

### 5.1 顶部减重

- 顶部只保留 `Back / 标题 / More`
- 模式切换器保留，但缩成更轻的分段控件
- 总高度明显低于上一轮

### 5.2 首屏不再堆功能入口

- 去掉正文前常驻的大按钮组
- 正文在模式切换后立刻进入主视觉区
- 用户第一眼先看见章节与内容，而不是功能卡片

### 5.3 功能改为按需进入

- 目录只有在需要跳转时打开
- `My Study` 只有在需要回看沉淀时打开
- `Details` 只有在需要看当前段落上下文时打开

### 5.4 状态轻量贴边

- 恢复提示不再长期挡正文
- 底部状态条只保留最小必要状态
- 选中句子的操作条仅在选中时出现，操作完成后退场

## 7. 当前结论

这轮移动端重构的核心不是“功能变少了”，而是“功能从正文前退后了”。

用户在这版里的感受应该更接近：

`我先开始读，只有在需要时才拉出结构、查看沉淀、打开当前段落详情。`

而不是：

`我先面对一组功能入口，然后再开始读。`
