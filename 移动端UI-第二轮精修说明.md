# 移动端 UI 第二轮精修说明

## 本轮删掉了哪些说明型文案

- 去掉了 `Details` 中解释设计逻辑的句子，改为真实段落摘录与真实状态。
- 去掉了 `My Study` 中“Review marked sentences...”这类功能说明句，改为数量与内容预览。
- 去掉了 `Structure` 中偏“降级导航说明”的写法，改为更接近目录的章节名 + 次级位置信息。
- 去掉了恢复提示中的说明式动作文案，改为更短、更像真实产品的状态提示。

## 哪些地方增强了真实内容感

- 首屏正文替换为更接近书籍阅读的拟真英文内容，不再出现“原文模式为什么这样设计”的说明句。
- `Structure` 优先呈现章节/小节名称，再附 `Page + Paragraph range` 作为次级信息。
- `My Study` 现在更像真实沉淀入口：
  - `12 highlights`
  - `6 tags`
  - `Last read`
  - 并带有简短内容预览
- `Details` 现在更像当前段落详情，而不是功能解释面板。

## 哪些元素进一步减重了

- 首屏章节标题层级继续压低，章节信息仍保留，但正文权重更高。
- 正文高亮进一步收紧为更贴近行内划线的轻标记，减少“黄色胶囊条感”。
- 临时底部操作条压缩了间距、按钮重量和整体厚度，更像短时 action bar。
- 恢复阅读提示变得更薄、更短、更轻。
- 底部状态条继续弱化按钮感，让状态更像状态标签，而不是底部操作按钮。

## 哪些地方继续保持与 PC 端一致

- `Structure` 仍然是文档结构导航，只是移动端用 bottom sheet 承载。
- `My Study` 仍然是我的沉淀入口，用于回看 `Highlights / Tags / Last read`。
- `Details` 仍然对应 PC 端右侧的当前段落详情面板，只是移动端改为底部弹层。
- 选中句子的临时操作仍然只在需要时出现，语义与 PC 端一致，只是承载方式改为底部轻操作条。
- 底部状态仍然只负责全局轻状态展示，不承担正文主交互。

## 当前仍保留到下一轮的问题

- 如果后续能从 PDF 提取更稳定的原始目录，`Structure` 还可以从当前的第一阶段降级目录进一步升级为更完整的书籍目录。
- 当前稿件已经更接近真实产品，但真机上的滚动手感、长按选中文本和浏览器底栏冲突，还需要在前端实现里继续做实机校准。
- 正文高亮已经更自然，但真正落地到代码后，仍建议按实际字体渲染效果再细调一次高亮贴合度。
