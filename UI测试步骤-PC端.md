# UI测试步骤-PC端

## 1. 打开方式

打开项目根目录下的 `UI设计-PC端`，依次查看：

1. `01-PC主阅读页-原文模式-连续正文版.png`
2. `02-PC主阅读页-译文模式-连续正文版.png`
3. `03-PC当前选中句子的浮层交互态.png`
4. `04-PC右侧状态变化稿.png`
5. `05-PC恢复阅读轻提示态.png`
6. `06-PC缓存完成与离线轻状态态.png`

## 2. 主阅读页检查

### 原文模式

检查 `01-PC主阅读页-原文模式-连续正文版.png`：

- 顶部两个模式按钮是否都清晰可见
- `原文模式` 是否处于当前选中态
- 中间主阅读区是否只显示英文正文
- 正文是否为连续阅读流，而不是段落卡片列表
- 当前段落是否只是轻选中，而不是大面积组件盒子
- 黄色高亮是否像正文内标注，而不是独立功能条
- 左侧栏是否减重，没有压过正文
- 右侧栏是否围绕当前段落组织信息

判定失败的典型情况：

- 正文被拆成多个大卡片
- 原文模式里混入中文正文
- 当前段落被做成强组件容器

### 译文模式

检查 `02-PC主阅读页-译文模式-连续正文版.png`：

- `译文模式` 是否处于当前选中态
- 每个阅读块是否为“一段原文 + 一段译文”上下结构
- 整体是否仍然保持连续阅读感
- 是否错误做成左右双栏
- 译文是否像段后辅助内容，而不是第二阅读列

判定失败的典型情况：

- 左右双栏对照
- 译文模式重新回到卡片列表
- 原文和译文被拆成两个并列系统区

## 3. 状态与交互检查

### 选中句子浮层

检查 `03-PC当前选中句子的浮层交互态.png`：

- 浮层是否只在选中句子时出现
- 浮层是否贴近选区
- 是否包含 `Highlight / Add tag / Copy`
- 浮层是否足够轻，不抢正文注意力

### 右侧详情面板

检查 `04-PC右侧状态变化稿.png`：

- 右侧是否固定承担“当前段落详情”角色
- 是否包含：
  - 段落位置
  - 翻译状态
  - 高亮与标签摘要
  - 必要轻操作
- 翻译失败与 `Retry translation` 是否在面板内表达
- 标签编辑是否已经从“整页状态”收回到局部变化

### 恢复阅读轻提示

检查 `05-PC恢复阅读轻提示态.png`：

- 恢复阅读是否通过顶部轻提示表达
- 提示是否足够短，不遮挡正文入口太久
- 是否使用真实动作文案，如 `Continue reading`、`Start from beginning`

### 缓存 / 离线 / 保存状态

检查 `06-PC缓存完成与离线轻状态态.png`：

- 是否通过轻提示或底部状态系统表达
- 是否清晰出现 `Cached / Available offline / Saved locally`
- 是否没有被做成独立整页专题图

## 4. 第一阶段边界复核

以下任一项出现，应判定不通过：

- 出现第二阶段能力或学习平台化模块
- 原文模式不再是纯英文正文
- 译文模式不是上下结构
- 恢复、翻译失败、缓存状态被做成独立大页面
- 右侧详情面板脱离“当前段落上下文”

## 5. 重点关注错误项

- 正文是否仍有被卡片化的趋势
- 右侧栏是否还存在明显“设计说明文案”
- 译文模式是否被误读成双栏模式
- 恢复阅读提示是否过重
- 浮层是否太大，反而抢走正文视觉中心
