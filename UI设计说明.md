## 2. 文档收敛结论

### 2.1 产品定位

- 这是英文原版 PDF 阅读辅助工具。
- 不是通用 PDF 阅读器。
- 不是学习平台。
- 不是社交产品。

### 2.2 第一阶段主链路

第一阶段 UI 只服务：

`导入 -> 阅读 -> 翻译 -> 标注 -> 恢复 -> 离线阅读`

### 2.3 第一阶段 UI 红线

- 必须同时覆盖 PC 端与移动端。
- 必须保留两个清晰按钮：`原文模式`、`译文模式`。
- `原文模式` 正文区只显示英文正文。
- `译文模式` 只能做“一段原文 + 一段译文”上下结构。
- 不允许左右双栏对照。
- 不允许把移动端做成缩小版 PC 三栏。
- 不允许把阅读页拆成后台系统式卡片列表。

## 3. 本轮修改的核心方向

本轮基于上一版稿件继续修正，重点纠正“阅读结构不对”的问题。

### 3.1 从段落卡片列表改回连续正文

- 主阅读区改为连续正文流。
- 段落之间只靠自然段间距与细分隔控制节奏。
- 去掉大面积段落卡片。
- 让正文恢复“数字书页”阅读感。

### 3.2 当前段落改为轻选中

- 当前段落使用左侧细竖线 + 轻选中信息标识。
- 不再用大面积蓝色块包住整段。
- 句子高亮直接发生在正文内部。

### 3.3 状态不再做成整页专题图

以下状态都改回阅读页内部变化：

- 当前选中句子的浮层交互态
- 右侧状态变化稿
- 恢复阅读轻提示态
- 缓存完成 / 离线轻状态态
- 移动端底部轻操作态
- 移动端底部弹层态

## 4. 原文模式说明

- 主阅读区为连续英文正文。
- 译文不进入正文主流。
- 当前段落有轻选中提示。
- 当前句子在正文内部被低饱和黄色高亮。
- 操作通过选区附近浮层或移动端轻操作条触发，不常驻占位。

## 5. 译文模式说明

- 仍然保持连续阅读感。
- 每个段落组采用“原文在上、译文在下”。
- 原文与译文属于同一阅读流，不拆成第二列。
- 更接近“段后辅助译文”，而不是卡片化模块。

## 6. PC 端布局说明

- 顶部工具栏保留品牌、文档名、导入 PDF、模式切换、搜索、更多。
- 左侧栏保留当前文档、Structure、My Study，但弱化大卡片感。
- 中间正文区改成连续书页阅读面。
- 右侧栏保留 `Current Paragraph / Translation Status / Highlights & Tags` 三块，但更像当前阅读上下文，而不是解释性小组件集合。
- 恢复阅读、缓存完成、选区操作等状态全部改为阅读页内部的轻提示或局部变化。

## 7. 移动端布局说明

- 顶部保留返回、文档名、模式切换、更多。
- 正文严格采用单列连续阅读流。
- 原文模式只显示英文正文。
- 译文模式仍然保持“原文在上、译文在下”。
- 目录与段落信息通过底部弹层 / 入口进入。
- 选区操作通过底部轻操作条展示，不常驻。

## 8. 与第一阶段需求的对应关系

- 导入：PC 顶部保留 `导入 PDF`。
- 阅读：双端主阅读区都以连续段落阅读为核心。
- 翻译：双端都有 `译文模式`，且是上下结构。
- 标注：句子高亮、Add tag、Copy 都保留。
- 恢复：改为阅读页内轻提示。
- 离线：通过状态栏与轻提示表达 `Cached / Available offline / Saved locally`。
