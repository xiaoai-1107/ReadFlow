# UI测试步骤-移动端

## 1. 打开方式

打开项目根目录下的 `UI设计-移动端`，依次查看：

1. `01-移动端主阅读页-原文模式-连续正文版.png`
2. `02-移动端主阅读页-译文模式-连续正文版.png`
3. `03-移动端句子选中后轻操作态.png`
4. `04-移动端标签与当前段落信息底部弹层态.png`
5. `05-移动端恢复阅读轻提示态.png`
6. `06-移动端离线可读状态提示态.png`

## 2. 主阅读页检查

### 原文模式

检查 `01-移动端主阅读页-原文模式-连续正文版.png`：

- 页面是否为手机竖屏单列阅读流
- 是否保留 `原文模式 / 译文模式` 两个按钮
- `原文模式` 是否为当前选中态
- 正文区是否只显示英文正文
- 首屏是否没有大段模式说明文案
- 段落之间是否只是轻分隔，不像列表项
- 是否没有保留 PC 式左右侧栏

### 译文模式

检查 `02-移动端主阅读页-译文模式-连续正文版.png`：

- `译文模式` 是否为当前选中态
- 是否仍为单列连续阅读流
- 每个段落块是否为“一段原文 + 一段译文”上下结构
- 是否没有做成左右并排
- 详情入口是否统一为 `Details`

判定失败的典型情况：

- 译文模式仍带有解释型标题
- 页面保留 PC 式侧栏思路
- 段落重新变成大卡片容器

## 3. 交互与状态检查

### 句子选中后的临时操作条

检查 `03-移动端句子选中后轻操作态.png`：

- 底部操作条是否只在选中句子时出现
- 是否包含 `Highlight / Add tag / Copy`
- 是否为临时轻量条，而不是常驻工具栏

### 当前段落详情底部弹层

检查 `04-移动端标签与当前段落信息底部弹层态.png`：

- 是否通过底部弹层承载当前段落详情
- 结构是否与 PC 右侧详情面板对应
- 是否包含：
  - 段落位置
  - 翻译状态
  - 高亮与标签摘要
  - 必要轻操作
- 入口是否统一，不再出现多种不同叫法

### 恢复阅读轻提示

检查 `05-移动端恢复阅读轻提示态.png`：

- 提示是否足够轻且靠近顶部
- 是否不会长时间占用正文首屏
- 是否使用真实动作文案，如 `Continue reading`

### 离线可读状态提示

检查 `06-移动端离线可读状态提示态.png`：

- 是否通过轻提示表达 `Available offline`
- 是否没有做成独立整页状态页
- 底部状态条是否更薄，不抢正文高度
- 是否同时保留底部全局状态信息

## 4. 第一阶段边界复核

以下任一项出现，应判定不通过：

- 移动端不是单列阅读流
- 原文模式混入中文正文
- 译文模式不是上下结构
- 出现 PC 式左右侧栏
- 选中操作变成常驻重工具栏
- 缓存、恢复、翻译失败被做成独立专题页

## 5. 重点关注错误项

- 首屏是否仍残留展示稿式解释文案
- `Details` 入口与底部弹层的语义是否统一
- 底部状态条是否仍然过厚
- 轻提示是否挡住正文
- 译文模式是否仍然像“说明稿”而不像真实阅读器
