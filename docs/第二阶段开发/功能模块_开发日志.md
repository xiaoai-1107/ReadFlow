# 第二阶段开发日志

> 开发日期：2026-04-09

## 1.1 句子级翻译增强

**状态：已完成**

### 开发内容

在第一阶段已实现 `ensureSentenceTranslation` 的基础上，在句子卡片（sentence card）的译文区域新增**重试按钮**，供用户在句子翻译失败时直接重试，无需刷新页面或关闭卡片。

### 实现细节

- 文件：`src/pages/ReaderView.vue`
- 新增函数：`retrySentenceTranslation()`，调用 `readerStore.ensureSentenceTranslation(sentenceId, true)`
- 模板：在句子卡片译文区增加 `v-else-if` 分支，当 `selectedSentenceTranslation?.status === 'failed'` 且可重试时显示重试按钮
- UI 文案：新增 `retrySentenceTranslation`、`sentenceTranslationRetried`（双语）

### 回归影响

- 不影响段落级翻译逻辑
- 句子级翻译与段落级翻译使用独立存储（`sentenceTranslations` vs `translations`），互不覆盖
- 用户可以在"Original"或"Translation"模式下独立查看句子翻译结果

## 1.2 查词能力

**状态：已完成**

### 开发内容

替换原有静态词卡（仅显示音标占位符）为动态查词功能：

1. 新增 `WordLookupRecord` 类型：包含 `word`、`contextSentenceId`、`definition`、`status`（loading/loaded/failed）等字段
2. 新增 `WordLookupInput` 接口及 `lookupWordInContext()` 服务函数：使用现有翻译端点发送含上下文的查词请求
3. 新增 `src/stores/wordLookup.ts` store：管理查词缓存，支持按 `word:sentenceId` 键去重
4. 查词自动在用户点击单词时触发（watch `activeWord`）
5. 词卡新增：加载中提示、失败时重试按钮、"收藏单词"/"取消收藏"按钮

### 缓存机制

- 查词结果写入 `localforage` 的 `word-lookups` store，以 documentId 为键分组
- 同一单词在同一句中只查询一次（除非 `force=true` 重试）
- 跨会话持久化

### 遇到的问题

- 原有词卡使用同一句子翻译作为词义来源，Phase 2 改为独立的查词请求。为兼容无翻译服务场景，失败时仍回落到句子翻译（`activeWordMeaning`）作为提示

## 1.3 单词收藏与单词本

**状态：已完成**

### 开发内容

1. 新增 `VocabRecord` 类型：包含 word、definition、exampleSentence、documentId、documentTitle、paragraphId、sentenceId、createdAt
2. 新增 `VocabBundle` 类型（全局存储容器）
3. 新增 `src/stores/vocab.ts` store：
   - `addEntry()` 支持去重（同 word + sentenceId 只保留一条）
   - `removeEntry()` 删除条目
   - `isWordSaved()` 判断是否已收藏
   - `sortedEntries` getter：按添加时间降序排列
4. 存储層新增 `getVocabBundle` / `saveVocabBundle` 函数，使用全局键（跨文档）
5. 词卡中新增"收藏单词"/"取消收藏"按钮（已收藏时高亮显示）
6. Study Panel 新增"单词本"标签页，展示全部已收藏单词及来源信息，支持逐条删除

## 1.5 导出"我的重点摘录"

**状态：已完成**

### 开发内容

1. 新增 `exportHighlights(format, tagFilter)` 函数，支持：
   - `format: 'markdown' | 'txt'`
   - `tagFilter: string | null`（按标签筛选，null 表示导出全部）
2. `buildExportMarkdown()` / `buildExportTxt()`：构建带段落位置、高亮文本、标签元数据的导出内容
3. `triggerDownload()`：使用 `Blob` + 临时 anchor 触发浏览器下载，无服务器依赖
4. Study Panel → Highlights 标签页：顶部新增"导出为 Markdown"/"导出为 TXT"按钮（全部高亮）
5. Study Panel → Tags 标签页：当前选中标签下方新增同款导出按钮（按标签筛选）
6. 文件名格式：`{文档标题slug}-highlights-{日期}.md/txt`

### 遇到的问题

- 导出内容需动态解析段落位置，通过 `paragraphs` 数组查找 `paragraphId` 对应的页码和段落序号
