# 第一阶段高亮与恢复专项记录

## 目标

保证高亮、标签、当前段落详情、恢复阅读都建立在稳定文本结构之上，并且双端行为一致。

## 第一阶段边界

允许：

- 段落级高亮
- 句子级高亮
- 标签绑定到高亮
- 从高亮/标签恢复到当前段落上下文

不做：

- 跨段自由选区高亮
- 画布式批注
- 独立于高亮存在的标签系统

## 当前高亮模型

### 段落高亮

- 目标：帮助标记重要段落或当前学习片段
- 绑定：`paragraphId`

### 句子高亮

- 目标：服务句子阅读与标记
- 绑定：`sentenceId`

### 标签

- 标签绑定到高亮记录
- 同一标签可关联多个高亮

## 当前恢复策略

### 1. 当前段落检测

使用 `IntersectionObserver` 追踪阅读区中的当前段落。

原因：

- 比 `scrollY` 更稳定
- 更适合双端布局
- 更贴合连续正文阅读流

### 2. 会话恢复

持久化：

- `documentId`
- `lastReadParagraphId`
- `mode`

恢复顺序：

- 恢复文档
- 恢复模式
- 恢复当前段落

### 3. 选中句子操作

PC：

- 句子选中后显示选区附近轻浮层

移动端：

- 句子选中后显示临时底部操作条

一致规则：

- 只在选中时出现
- 取消选中后消失
- 不常驻

## 当前已落地的详情承载

PC：

- 右侧当前段落详情面板

移动端：

- 当前段落详情底部弹层

内容结构一致：

1. 当前段落位置
2. 翻译状态
3. 高亮与标签摘要
4. 必要轻操作

## 2026-03-18 落地结论

当前高亮与恢复主链已经接入真实数据：

- 高亮可持久化
- 标签可持久化
- 重进文档可恢复阅读位置与模式
- 当前段落详情在双端消费同一套底层状态

## 当前遗留

- 当前“句子选中”实现偏保守，更接近句子级选择，不是浏览器原生任意文本 range 选区
- 真正的浏览器手势与移动端长按选区体验，仍需下一轮真机联调继续优化

## 2026-03-18 移动端专项补充

为强化移动端“临时操作条”语义，本轮补充了以下行为约束：

1. 句子选中后的底部操作条仍只在选中状态出现。
2. 在移动端执行 `Highlight`、`Add tag`、`Copy` 后，会自动清除当前选中态并收起临时操作条。
3. 这样做的目的是避免操作完成后底部条继续停留，削弱阅读主线。
