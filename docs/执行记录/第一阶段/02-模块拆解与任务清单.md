# 模块拆解与任务清单

## 1. 使用方式

本文件按模块拆解第一阶段任务。每个模块均给出目标、输入、输出、子任务、风险与验收点，供后续逐模块执行与记录。

## 2. 模块清单

### M01 文档导入与删除

- 目标：建立文本型 PDF 的导入、重复校验、删除与清理闭环。
- 输入：用户选择的 PDF、本地文件元信息、导入约束。
- 输出：可持久化的文档记录与可回滚的导入结果。
- 子任务：
  - 接入文件选择与导入入口。
  - 校验文件格式、大小、文本层可解析性。
  - 为文档生成稳定 `documentId`。
  - 删除文档时联动清理解析结果、缓存、会话、高亮、标签。
- 风险：重复导入识别不稳定；删除后残留脏数据。
- 验收点：导入成功、重复导入提示、删除后关联数据不残留。

### M02 文档列表与切换

- 目标：提供文档库视图、最近阅读信息与切换入口。
- 输入：文档元信息、最近会话、缓存可用性。
- 输出：列表页、筛选/排序结果、当前文档选择状态。
- 子任务：
  - 展示标题、导入时间、最后阅读时间、缓存状态。
  - 支持进入阅读、返回列表、切换文档。
  - 空状态、异常状态提示。
- 风险：文档删除后当前选择状态未同步。
- 验收点：列表展示稳定，切换后路由与状态一致。

### M03 PDF 解析

- 目标：从文本型 PDF 中提取可标准化文本与页级上下文。
- 输入：原始 PDF 文件。
- 输出：解析文本块、页码映射、解析元数据。
- 子任务：
  - 接入 PDF 文本层提取。
  - 记录页码、原始块顺序、基础位置信息。
  - 建立解析失败分类与提示。
- 风险：扫描件误判为可解析；块顺序错乱。
- 验收点：文本型 PDF 可稳定提取文本；无文本层时给出明确提示。

### M04 文本标准化

- 目标：把解析噪声统一收敛为可重复的标准文本。
- 输入：原始文本块、页级上下文、标准化规则。
- 输出：标准化文本流与规则版本号。
- 子任务：
  - 清理多余换行、连字、页眉页脚噪声、重复空白。
  - 保留必要标点、编号、缩写上下文。
  - 为标准化规则建立版本标识。
- 风险：过度清洗导致语义损失；规则变动引起锚点漂移。
- 验收点：同文档重复标准化结果一致；规则可追溯。

### M05 段落生成

- 目标：生成第一阶段主阅读单元。
- 输入：标准化文本流、段落规则。
- 输出：稳定段落列表、段落 ID、页码范围映射。
- 子任务：
  - 设计段落切分规则。
  - 生成段落顺序号、锚点、前后邻接关系。
  - 校验重复解析稳定性。
- 风险：段落边界与显示边界不一致。
- 验收点：段落成为阅读、翻译、恢复、缓存的统一主单元。

### M06 句子辅助单元生成

- 目标：在段落内部生成稳定句子辅助单元，服务高亮、局部定位与恢复。
- 输入：段落文本、句子切分规则。
- 输出：句子列表、句子 ID、段落内偏移映射。
- 子任务：
  - 设计不能简单按句号切分的规则。
  - 处理缩写、数字、小数、项目编号、引号、括号等情形。
  - 建立句子切分稳定性测试样本。
- 风险：句子边界随渲染或标准化变化漂移。
- 验收点：同段落重复切分稳定；高亮与恢复都复用该结果。

### M07 阅读视图

- 目标：渲染段落主视图与段落内句子辅助信息。
- 输入：文档、段落、句子、当前模式、翻译数据。
- 输出：阅读页渲染结构。
- 子任务：
  - 建立阅读页主结构与滚动容器。
  - 支持原文模式、双语模式等第一阶段模式。
  - 以段落为主组织 UI，以句子为辅助定位。
- 风险：渲染阶段重新切分文本。
- 验收点：UI 与存储共用同一结构边界。

### M08 阅读模式切换

- 目标：在不破坏锚点与恢复前提下切换展示模式。
- 输入：阅读模式定义、当前阅读落点。
- 输出：模式状态、模式切换行为。
- 子任务：
  - 定义第一阶段模式集合。
  - 模式切换时保留当前段落锚点。
  - 记录模式切换到会话快照。
- 风险：切换模式后高亮或恢复落点漂移。
- 验收点：切换前后仍能定位到同一内容附近。

### M09 段落翻译

- 目标：建立段落翻译主链。
- 输入：段落锚点、段落文本、翻译 Provider。
- 输出：段落译文、翻译状态、重试入口。
- 子任务：
  - 设计翻译触发策略。
  - 用段落锚点绑定译文。
  - 去重请求、防重复翻译。
- 风险：按显示顺序贴译文导致错位。
- 验收点：同一段落译文稳定复用，不随重排错位。

### M10 翻译缓存

- 目标：持久化已获得的译文并支持恢复。
- 输入：翻译结果、段落锚点、缓存策略。
- 输出：译文缓存记录。
- 子任务：
  - 设计缓存键与版本策略。
  - 读写缓存、命中缓存、失效重译。
  - 离线恢复已缓存译文。
- 风险：译文缓存与段落边界不一致。
- 验收点：重开页面和离线场景下已缓存译文可准确恢复。

### M11 高亮系统

- 目标：实现第一阶段高亮主链。
- 输入：段落、句子、文本快照、用户选区。
- 输出：高亮对象、恢复定位信息。
- 子任务：
  - 明确第一阶段支持的高亮粒度。
  - 同时记录结构锚点与文本快照。
  - 支持新增、编辑、删除、恢复。
- 风险：仅靠字符偏移或仅靠显示顺序导致恢复失效。
- 验收点：跨重开、跨离线恢复仍能定位到正确内容附近。

### M12 标签系统

- 目标：让高亮具备分类、筛选与跳转能力。
- 输入：高亮对象、标签定义。
- 输出：标签列表、筛选结果、跳转结果。
- 子任务：
  - 标签绑定到高亮，而不是绑定到裸文本。
  - 支持标签筛选、列表展示、点击跳转回文。
  - 标签删除与高亮删除联动。
- 风险：标签与高亮失联；跳转锚点过期。
- 验收点：标签筛选与回文行为稳定。

### M13 阅读进度保存

- 目标：保存足以恢复阅读现场的最小必要状态。
- 输入：当前文档、模式、段落锚点、句子锚点、滚动微调信息。
- 输出：会话快照。
- 子任务：
  - 定义持久化字段。
  - 在切换文档、切换模式、离开页面等时机保存。
  - 与高亮/标签上下文保持兼容。
- 风险：保存过少导致恢复失败；保存过多导致状态耦合。
- 验收点：会话快照能独立支撑恢复链路。

### M14 会话恢复

- 目标：按稳定顺序恢复阅读现场。
- 输入：会话快照、文档结构、缓存数据。
- 输出：恢复结果、回退路径、提示信息。
- 子任务：
  - 文档级恢复。
  - 阅读模式恢复。
  - 段落锚点恢复。
  - 句子/文本快照微调。
  - 滚动位置辅助微调。
- 风险：恢复链路顺序错误；无锚点时回退不明确。
- 验收点：冷启动、热启动、离线恢复均可工作。

### M15 缓存与离线

- 目标：建立已读内容的离线可读链路。
- 输入：文档、段落原文、译文、高亮、标签、会话。
- 输出：缓存记录、离线可用性状态。
- 子任务：
  - 明确缓存范围与粒度。
  - 设计已读缓存优先策略。
  - 提供离线命中与未命中提示。
- 风险：缓存对象边界不一致；部分缓存导致空白显示。
- 验收点：已缓存内容离线可读，未缓存内容有可理解提示。

### M16 错误提示与状态反馈

- 目标：让导入、解析、翻译、高亮、恢复、缓存等主链失败都有明确反馈。
- 输入：各模块错误类型、任务状态。
- 输出：统一错误态与 UI 提示。
- 子任务：
  - 定义错误分类。
  - 统一 loading、success、empty、error、offline 状态表达。
  - 避免错误态污染主状态。
- 风险：业务状态与提示状态耦合。
- 验收点：任一主链失败都可理解、可恢复或可回退。

### M17 测试与验收

- 目标：通过模块测试、集成测试与专项回归保证第一阶段可交付。
- 输入：全部功能模块、风险清单、验收标准。
- 输出：测试记录、问题单、回归结果、验收结论。
- 子任务：
  - 建立模块级测试项。
  - 建立专项回归：翻译错位、句子稳定、高亮恢复、标签跳转、进度恢复、离线缓存。
  - 记录实测结果与问题闭环。
- 风险：只测 happy path；未覆盖恢复与离线边界。
- 验收点：风险清单均有对应测试与结论。
