## 3. 第一阶段功能边界收敛

结合产品需求、设计文档和执行记录，第一阶段主链保持不变：

`导入 -> 解析 -> 阅读 -> 原文/译文模式切换 -> 段落翻译 -> 段落/句子高亮 -> 标签 -> 阅读恢复 -> 已缓存内容离线可读`

### 3.1 第一阶段必须具备

1. PDF 导入与本地文档管理
2. 文本型 PDF 解析
3. 段落作为主阅读单元
4. 句子作为辅助阅读单元
5. PC 与移动端双端阅读
6. 原文模式与译文模式
7. 段落翻译与重试
8. 段落级高亮
9. 句子级高亮
10. 标签绑定与回看
11. Structure 结构导航
12. My Study 全局沉淀入口
13. 当前段落详情
14. 阅读位置与模式恢复
15. 已缓存内容离线可读
16. 明确的错误态、空态和状态提示

### 3.2 第一阶段明确不做

1. OCR / 扫描件 PDF 支持
2. 账号体系与云同步
3. 完整单词本
4. 正式查词系统
5. AI 总结 / 脑图
6. TTS
7. 导出学习材料
8. 自由任意文本 range 高亮

## 6. 当前版本的主要遗漏与不完全项

### 6.1 属于第一阶段范围内，但当前仅达到降级可用

#### 1. Structure 仍未达到“真实书籍目录”层级

当前情况：

1. 已不是正文 preview 列表
2. 已支持点击跳转与当前项联动
3. 但本质仍是“按页聚合的降级结构导航”

原因：

1. 当前解析器没有抽取 PDF 原生 outline / chapter tree
2. 当前 `structureItems` 由页级段落聚合推导而来，而非真实章节目录

结论：

- 这项已达到第一阶段可接受的降级可用
- 但若以“真实书籍目录导航”标准看，仍是部分完成

#### 2. 翻译能力已连通，但生产级翻译质量仍未闭环

当前情况：

1. 已支持外部 HTTP 翻译 endpoint
2. 未配置 endpoint 时使用 glossary fallback
3. fallback 结果会真实写回状态和缓存，不是假成功态

结论：

- 就“第一阶段功能链路可运行”而言，这项已可用
- 就“正式上线可接受的翻译质量”而言，这项仍是部分完成

### 6.2 当前仍缺少的不是功能代码，而是验收层补齐

#### 1. 真机/浏览器人工交互验收尚未完成

当前缺口：

1. 真机长按与句子选择体验
2. 长文滚动手感
3. 抽屉 / bottom sheet 高度策略
4. 真实浏览器操作下的连续阅读体验

说明：

- 这不是代码链路缺失
- 但它是“第一阶段完全验收”尚未闭环的地方

#### 2. 真实断网环境下的人工离线验收尚未完成

当前缺口：

1. 已缓存原文的断网阅读人工验证
2. 已缓存译文的断网阅读人工验证
3. 未缓存内容的断网边界提示人工验证

说明：

- 代码与状态链路已经存在
- 但尚未补完真实浏览器断网复测

## 8. 当前版本综合判断

### 8.1 已完成的程度

当前版本已经具备第一阶段主链所需的大部分真实能力：

1. 可以导入并管理本地 PDF
2. 可以解析文本型 PDF
3. 可以在 PC 和移动端进入连续阅读
4. 可以在原文模式与译文模式之间切换
5. 可以触发并缓存段落翻译
6. 可以做段落/句子高亮与标签
7. 可以从 My Study 和 tag 结果回到原文
8. 可以恢复阅读位置与模式
9. 可以在已缓存前提下离线阅读

### 8.2 当前版本是否存在第一阶段功能遗漏

结论：

1. 第一阶段主链没有出现“整项功能缺失”的重大遗漏
2. 当前更主要的问题不是“少了某个核心能力”
3. 而是以下三类收口问题仍存在：
   - `Structure` 仍是降级结构导航，不是真实章节目录
   - 正式外部翻译服务尚未接入
   - 真机与断网人工验收尚未补完

### 8.3 当前版本是否达到“第一阶段可实际使用”

结论：

- 从代码链路、状态持久化、构建结果和已有专项记录看，当前版本已达到“第一阶段可实际使用的开发版”
- 从正式验收角度看，仍需补齐浏览器/真机/断网人工验证，且上线前需要接入真实翻译 provider

## 9. 建议的下一步优先级

1. 补做浏览器与真机人工全链路验收
2. 补做真实断网环境离线验收
3. 如要提升导航质量，再做 PDF 真实目录提取或更强的章节识别
4. 如要进入上线准备，再接入真实翻译服务
