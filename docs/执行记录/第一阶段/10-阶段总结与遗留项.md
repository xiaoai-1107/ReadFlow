# 第一阶段阶段总结与遗留项

## 当前结论

截至 2026-03-18，本项目已经完成一版面向 PC 端与移动端的第一阶段重构开发版。

它具备以下特征：

- 不是静态演示页
- 不是只还原 UI 的原型
- 已具备真实数据、真实状态、真实持久化链路

## 已完成能力

1. 文档导入与本地文档库
2. 文本型 PDF 解析与段落化阅读结构
3. PC 多区域阅读布局
4. 移动端单列连续阅读流
5. 原文模式与译文模式
6. 当前段落与邻近段落的翻译触发
7. 翻译状态展示与 Retry
8. 段落/句子高亮
9. 标签创建与展示
10. 当前段落详情层
11. 阅读位置与模式恢复
12. 本地缓存与离线阅读基础能力
13. PWA 构建产物输出

## 未完全闭合但不阻断开发版使用的项

1. 未接入真实翻译 provider 时，使用 glossary fallback 作为降级翻译
2. 未完成真机浏览器手势与长按操作联调
3. 未完成复杂版式 PDF 的专项回归样本验证

## 当前阻断上线质量的项

1. 真实翻译服务未完成联调
2. 浏览器侧人工全链路验收未完成
3. 自动化 UI / E2E 测试体系未建立

## 第二轮优先事项

1. 接入并验证真实翻译 provider
2. 在桌面浏览器与移动端浏览器完成人工全链路测试
3. 为复杂 PDF 样本补解析回归
4. 进一步优化移动端选区体验和底部弹层高度策略

## 对下一阶段的明确边界

下一轮仍不建议直接跳到第二阶段功能扩张。

优先顺序应为：

1. 把第一阶段主链验证扎实
2. 把翻译质量、真机体验和解析稳定性补齐
3. 再考虑第二阶段扩展能力

## 2026-03-18 移动端专项补充结论

本轮移动端专项后，第一阶段移动端已经进一步接近“真实可用”而不是“响应式展示稿”：

1. 当前段落详情底部弹层的角色进一步固定。
2. 句子选中后的临时操作条不再有常驻化倾向。
3. 全局底部状态区对离线边界的表达更接近真实缓存链路。

当前仍需继续补齐的移动端问题主要集中在：

1. 真机长按、点击、滚动与弹层高度策略的人工体验验证。
2. 更细粒度的译文离线覆盖提示。
3. 复杂 PDF 样本下的移动端长文阅读回归。
