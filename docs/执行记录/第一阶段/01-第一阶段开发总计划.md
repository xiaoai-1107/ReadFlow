# 第一阶段开发总计划

## 1. 目标与版本边界

第一阶段目标是交付 ReadFlow MVP：围绕文本型 PDF 阅读，打通“导入 -> 解析 -> 标准化 -> 段落主阅读 -> 段落翻译 -> 句子级高亮/标签 -> 阅读进度恢复 -> 已读缓存与离线可读 -> 测试验收”的完整闭环。

版本边界：

1. 范围内：本地文档阅读、段落主链、句子辅助链、高亮与标签、恢复、缓存与离线、错误反馈、测试验收。
2. 范围外：OCR、扫描件、云同步、账号体系、多格式导入、复杂协作与推荐。

## 2. 总体执行顺序

执行顺序冻结如下：

1. Phase 0：前提冻结与基础约束收口。
2. Phase 1：文档导入与阅读主结构。
3. Phase 2：文本标准化、段落/句子单元生成。
4. Phase 3：段落翻译主链与翻译绑定。
5. Phase 4：高亮与标签。
6. Phase 5：阅读进度保存与恢复。
7. Phase 6：缓存与离线链路。
8. Phase 7：错误状态、联调、测试、验收与清理。

## 3. 关键依赖关系

关键依赖如下：

1. `Phase 2` 依赖 `Phase 1` 完成可稳定解析的文档导入主链。
2. `Phase 3` 依赖 `Phase 2` 输出稳定段落锚点与一致文本边界。
3. `Phase 4` 依赖 `Phase 2` 的句子辅助单元和 `Phase 3` 的段落绑定规则。
4. `Phase 5` 依赖 `Phase 2-4` 已冻结阅读单元、交互单元与状态模型。
5. `Phase 6` 依赖 `Phase 3-5` 已明确缓存对象、恢复对象与离线边界。
6. `Phase 7` 覆盖所有前置 Phase 的集成与回归。

## 6. 风险总览

最高优先级风险：

1. 原文与译文错位。
2. 句子切分不稳定。
3. 高亮位置恢复失败。
4. 标签跳转失效。
5. 页面关闭后阅读位置恢复错误。
6. 离线状态下内容空白。
7. 已缓存内容无法正确显示。
8. 同一内容重复切分、重复翻译、重复缓存。
9. 显示文本、翻译文本、缓存文本边界不一致。
10. 主状态与派生状态混杂导致行为异常。

## 7. 总体验收标准

总体验收以以下条件同时满足为准：

1. 第一阶段范围内功能全部落地，且与 `产品需求.md` 无冲突。
2. 阅读主链、翻译主链、高亮标签链、恢复链、缓存离线链全部贯通。
3. 关键回归风险均有专项测试记录。
4. 开发日志、问题单、决策记录、测试记录、阶段总结完整可追溯。
