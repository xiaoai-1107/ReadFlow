# 第三阶段开发任务：同步边注式学习笔记与 PDF 导出

## 1. 阶段定位

第三阶段的核心目标，是将 ReadFlow 从“英文文章阅读 + 翻译辅助 + 高亮标签”升级为“英文精读 + 同步边注 + 复习导出”的学习闭环。

本阶段重点实现：

- 句子右侧同步边注式学习笔记
- 从已选句子或高亮句子创建笔记
- 笔记与原文句子位置同步
- 笔记正文、翻译、标签编辑
- 笔记栏隐藏、展开和宽度拖拽
- 当前文档学习笔记 PDF 导出

本阶段不扩展复杂 AI、云同步、社交分享、协作批注和付费体系。优先保证阅读、记录、复习三件事稳定、清晰、可用。

---

## 2. 现状基线与修正结论

对照当前项目代码后，第三阶段开发应基于已有能力扩展，而不是另起一套阅读、解析或存储体系。

### 2.1 已存在能力

当前项目已经具备：

- `ParagraphUnit.sentences`：PDF 解析后已经有句子级数据。
- `SentenceUnit.id / index / anchorKey / textHash`：句子已有稳定 ID、顺序、锚点和文本指纹。
- `SentenceTranslationRecord`：已有句子级翻译缓存。
- `HighlightRecord`：已有段落和句子高亮。
- `TagRecord`：已有标签，并通过 `highlightIds` 绑定到高亮。
- `localforage`：当前本地存储已经使用 localforage，而不是 Dexie。
- Markdown / TXT 高亮导出：已有导出下载机制，可作为 PDF 预览导出的基础。


## 3. 阶段目标

完成后，用户可以在阅读英文文章时选中重点句子，创建学习笔记，并在原句右侧看到对应的翻译、标签和个人理解。用户可以编辑、删除、隐藏和导出这些笔记。

目标学习流程：

```text
导入英文文档
↓
阅读原文
↓
选中或高亮重点句子
↓
创建同步边注笔记
↓
查看或修改句子翻译
↓
添加标签和个人理解
↓
导出 PDF 学习笔记
↓
复习 / 打印 / 归档
```

## 4. 页面结构调整

### 4.1 当前结构

当前桌面端阅读页大致为：

```text
左侧目录 / 学习入口 + 中间原文阅读区 + 右侧当前段落 / 翻译 / 标签详情栏
```

### 4.2 第三阶段目标结构

第三阶段桌面端建议调整为：

```text
左侧目录 / 学习入口 + 中间阅读工作区

中间阅读工作区 =
原文句子列 + 可拖拽分隔条 + 同步边注笔记列
```

关键点：

- 同步边注不是一个普通右侧列表，而是阅读流的一部分。
- 每个句子行包含原文句子区域和笔记区域。
- 右侧原有详情栏能力应合并到边注笔记卡、浮层或学习面板中，避免桌面端出现两个竞争性的右侧栏。
- 小屏幕下默认隐藏边注列，使用底部抽屉展示当前句子的笔记。

### 4.3 推荐 DOM 结构

第一版优先采用句子行级布局：

```html
<div class="reader-workspace">
  <div class="sentence-row">
    <div class="sentence-text">
      English sentence
    </div>

    <div class="sentence-note">
      translation, tags, personal note
    </div>
  </div>
</div>
```

不建议第一版使用：

```text
PDF 页面坐标 + 绝对定位 + 滚动监听
```

原因：

- 实现复杂。
- 缩放后容易错位。
- 和现有流式阅读排版冲突。
- 小屏幕适配成本高。

## 5. 功能一：同步边注式笔记列

### 5.1 功能说明

在阅读工作区右侧增加同步边注列。笔记显示在对应英文句子的右侧，并随原文一起滚动。

### 5.2 交互要求

- 有笔记的句子右侧显示笔记卡片。
- 无笔记的句子右侧保持空白占位，不显示空卡片。
- 原文滚动时，笔记自然跟随对应句子滚动。
- 用户点击笔记卡片时，可以进入编辑态。
- 用户点击原文句子时，可以打开句子操作浮层。

### 5.3 预期效果

```text
英文原文                                      同步边注

To my wife, Cynthia, who told me              中文翻译：
to take the job at Stanford...                献给我的妻子 Cynthia……

                                              标签：
                                              #表达积累 #人生选择

                                              我的笔记：
                                              told me to do sth. 可以积累。
```

### 5.4 验收标准

- 笔记始终显示在对应句子右侧。
- 滚动阅读时笔记不脱离对应原句。
- 页面缩放或窗口变化时，笔记没有明显错位。
- 长笔记不会把正文挤压到不可读。

## 6. 功能二：从句子创建学习笔记

### 6.1 功能说明

用户选中句子或点击句子后，可以创建学习笔记。创建笔记时应复用现有句子选择、高亮、句子翻译和标签能力。

### 6.2 推荐交互

用户选中句子后，句子操作浮层增加“加入笔记”：

```text
高亮 | 翻译 | 加入笔记 | 复制 | 标签
```

点击“加入笔记”后：

- 若该句没有高亮，先创建句子高亮。
- 若该句没有笔记，创建一条 `NoteRecord`。
- 若该句已有笔记，打开已有笔记，而不是重复创建。
- 若已有句子翻译，自动带入翻译。
- 若没有句子翻译，允许先创建空翻译字段，并在翻译完成后补齐。

### 6.3 去重规则

第一版应保证：

```text
同一 documentId + sentenceId 只能有一条主笔记
```

后续如果需要同一句多条卡片，再单独设计多笔记模型。

### 6.4 验收标准

- 可以从选中句子创建笔记。
- 同一句子不会重复生成多条相同笔记。
- 创建笔记后原文句子有高亮或标记。
- 删除笔记后，如果该高亮仅为笔记服务，应同步移除高亮。
- 如果高亮仍被其他标签或学习功能使用，应谨慎处理，不应误删用户已有标记。

## 7. 功能三：笔记编辑

### 7.1 功能说明

笔记卡片支持查看和编辑以下内容：

- 英文原句
- 中文翻译
- 标签
- 用户个人笔记
- 来源位置

### 7.2 第一版编辑方式

第一版不做复杂富文本，采用普通表单：

- `textarea` 编辑个人笔记。
- `textarea` 或普通文本框修改翻译。
- `input` 输入标签。
- 回车添加标签。
- 点击标签关闭按钮删除标签。
- 保存按钮持久化修改。
- 删除按钮删除整条笔记。

### 7.3 标签处理规则

标签应优先复用现有 `TagRecord`：

- 笔记创建时关联一个句子高亮。
- 标签继续通过 `TagRecord.highlightIds` 绑定到该高亮。
- 笔记卡片展示标签时，根据 `highlightId` 反查标签。
- 导出时再把标签名称聚合为展示文本。

这样可以避免第三阶段新建一套和现有高亮标签并行的数据结构。

### 7.4 验收标准

- 可以编辑个人笔记。
- 可以修改翻译。
- 可以新增和删除标签。
- 保存后刷新页面数据不丢失。
- 删除笔记后边注卡片消失。
- 删除笔记不会误删同文档其他标签数据。

## 8. 功能四：笔记栏隐藏、展开和宽度拖拽

### 8.1 功能说明

用户可以隐藏或展开同步边注列，也可以拖拽调整边注列宽度。

### 8.2 推荐宽度

```text
最小宽度：280px
默认宽度：360px
最大宽度：520px
```

### 8.3 交互要求

- 点击隐藏按钮后，边注列收起。
- 原文阅读区自动获得更多宽度。
- 再次点击后，边注列恢复。
- 鼠标移动到分隔条时显示可拖拽光标。
- 拖拽过程中实时改变边注列宽度。
- 拖拽结果保存到本地设置。
- 小屏幕下默认隐藏边注列。

### 8.4 状态字段建议

扩展现有 `UiSettings`，而不是新建独立设置来源：

```ts
interface UiSettings {
  readerUiLanguage: UiLanguage
  notePanelVisible: boolean
  notePanelWidth: number
  updatedAt: number
}
```

### 8.5 验收标准

- 可以正常隐藏和恢复边注列。
- 拖拽体验流畅。
- 刷新页面后保留上一次的显示状态和宽度。
- 拖拽不影响正文选词、句子选择和高亮操作。
- 小屏幕下正文仍然可读。

## 9. 功能五：学习笔记 PDF 导出

### 9.1 功能说明

当前文档的学习笔记可以导出为 PDF。导出的 PDF 不包含完整原文页面，只包含用户标记过的学习内容。

### 9.2 导出内容

导出内容包括：

- 文档名称
- 导出时间
- 笔记数量
- 英文原句
- 中文翻译
- 标签
- 用户个人笔记
- 来源页码 / 段落 / 句子位置

### 9.3 第一版实现方式

第一版采用：

```text
导出预览页 + 浏览器打印保存 PDF
```

推荐流程：

```text
读取当前文档 NoteBundle
↓
按 pageIndex / paragraph order / sentence index 排序
↓
生成打印友好的导出预览页
↓
调用 window.print() 或提示用户使用浏览器打印
↓
用户保存为 PDF
```

实现时不要假定浏览器可以直接写入本地 `notes-export.html` 文件。可以使用以下方案之一：

- 新增 SPA 路由，例如 `/reader/:documentId/notes-export`。
- 打开新窗口并写入 Blob HTML。
- 在当前页打开打印预览模态，再调用浏览器打印。

### 9.4 PDF 示例结构

```text
Designing your life.pdf
学习笔记导出

导出时间：2026-07-02
笔记数量：32 条

--------------------------------

01 / 第 4 页 / 段落 23 / 句子 2

英文原句：
To my wife, Cynthia, who told me to take the job at Stanford...

中文翻译：
献给我的妻子 Cynthia，她劝我接受斯坦福的工作……

标签：
#好句 #人生选择

我的笔记：
这里可以积累 told me to do sth. 的表达。
```

### 9.5 版权边界

导出功能应避免把整本书内容重新打包：

- 只导出用户创建笔记的句子。
- 不导出完整原书页面。
- 不提供公开分享书籍内容的功能。
- 导出文件默认用于个人学习和复习。

### 9.6 验收标准

- 可以导出当前文档全部学习笔记。
- 导出顺序与原文顺序一致。
- 中文内容在浏览器打印 PDF 中不乱码。
- 空笔记时有明确提示，不生成空 PDF。
- 导出内容不包含完整原文页面。

## 10. 推荐数据结构

### 10.1 NoteRecord

新增笔记数据结构建议如下：

```ts
interface NoteRecord {
  id: string
  documentId: string
  documentTitle: string

  paragraphId: string
  paragraphAnchorKey: string
  paragraphOrder: number

  sentenceId: string
  sentenceAnchorKey: string
  sentenceIndex: number
  pageIndex: number

  highlightId: string | null

  originalText: string
  originalTextHash: string
  translationText: string
  translationUpdatedAt: number | null

  userNote: string

  createdAt: number
  updatedAt: number
}
```

说明：

- `pageIndex` 内部保持 0 基，界面和导出时显示为 `pageIndex + 1`。
- `highlightId` 用于复用现有高亮和标签能力。
- `originalTextHash` 用于判断原句内容是否与缓存笔记仍然匹配。
- 标签不建议第一版直接冗余到 `NoteRecord.tags`，展示和导出时从现有 `TagRecord` 聚合。

### 10.2 NoteBundle

延续当前按文档保存 bundle 的模式：

```ts
interface NoteBundle {
  documentId: string
  notes: NoteRecord[]
  updatedAt: number
}
```

### 10.3 派生展示模型

界面渲染时可以派生 `NoteViewItem`：

```ts
interface NoteViewItem extends NoteRecord {
  tags: TagRecord[]
}
```

该结构只用于 UI，不作为持久化主模型。

## 11. 本地存储建议

当前项目已经使用 `localforage`，第三阶段继续沿用。

建议在 `src/services/storage/index.ts` 增加：

```ts
notes: localforage.createInstance({ name: DB_NAME, storeName: 'notes' })
```

并提供以下存储接口：

```ts
getNoteBundle(documentId: string): Promise<NoteBundle>
saveNoteBundle(bundle: NoteBundle): Promise<void>
saveNoteRecord(record: NoteRecord): Promise<void>
deleteNoteRecord(documentId: string, noteId: string): Promise<void>
```

数据职责：

- `parsed`：保存文档解析结果和句子结构。
- `sentence-translations`：保存句子翻译。
- `annotations`：保存高亮和标签。
- `notes`：保存学习笔记正文和笔记元数据。
- `settings`：保存语言、边注显示状态和边注宽度。

## 12. 开发任务拆分

### 12.1 小阶段一：数据模型与存储

目标：

- 建立笔记数据模型。
- 增加本地笔记存储。
- 与现有句子、高亮、标签数据打通。

任务：

- 在 `src/domain/types.ts` 增加 `NoteRecord` 和 `NoteBundle`。
- 在 storage service 增加 `notes` store 和读写接口。
- 新增 `notes` Pinia store，负责加载、创建、编辑、删除笔记。
- 增加按 `documentId + sentenceId` 查找笔记的方法。
- 增加按原文顺序排序笔记的方法。

预期效果：

- 可以在本地保存和读取当前文档笔记。
- 刷新页面后笔记不丢失。
- 同一句子不会重复创建主笔记。

### 12.2 小阶段二：阅读布局改造

目标：

- 将阅读区改造成支持同步边注的工作区。

任务：

- 将桌面端阅读主体调整为“原文列 + 分隔条 + 边注列”。
- 保留左侧目录和学习入口。
- 将现有右侧详情能力迁移到边注卡片、句子浮层或学习面板。
- 为每个句子行预留边注区域。
- 保证当前阅读、翻译、搜索、段落定位功能不受影响。

预期效果：

- 桌面端可看到原文区和右侧同步边注区。
- 原有阅读功能仍然正常。
- 无笔记句子不会产生明显视觉噪音。

### 12.3 小阶段三：创建笔记

目标：

- 从句子选择或句子高亮创建学习笔记。

任务：

- 在句子浮层增加“加入笔记”操作。
- 创建笔记时复用当前 `SentenceUnit`、`SentenceTranslationRecord` 和 `HighlightRecord`。
- 没有高亮时自动创建句子高亮。
- 已有笔记时打开已有笔记。
- 创建后在对应句子右侧显示笔记卡片。

预期效果：

- 用户可以从阅读动作自然进入记笔记。
- 笔记与原句准确绑定。
- 创建笔记不会破坏现有高亮和标签流程。

### 12.4 小阶段四：笔记编辑与删除

目标：

- 完成笔记核心 CRUD。

任务：

- 创建 `NoteCard` 组件。
- 支持编辑个人笔记。
- 支持修改翻译文本。
- 支持新增和删除标签。
- 支持删除整条笔记。
- 删除笔记时处理对应高亮状态。

预期效果：

- 用户可以完成从划线到记录、编辑、删除笔记的完整流程。
- 数据刷新后仍然保持一致。
- 删除笔记不会产生孤立 UI 状态。

### 12.5 小阶段五：边注列隐藏和拖拽

目标：

- 优化阅读空间与笔记空间切换体验。

任务：

- 增加隐藏 / 展开按钮。
- 增加拖拽分隔条。
- 限制边注列最小 / 最大宽度。
- 将用户宽度和显示状态写入 `UiSettings`。
- 适配小屏幕默认隐藏。

预期效果：

- 用户可以自由切换纯阅读和边注阅读。
- 用户可以根据屏幕调整边注列宽度。
- 拖拽不会影响正文选择和句子点击。

### 12.6 小阶段六：PDF 导出

目标：

- 将当前文档学习笔记导出为可打印 PDF。

任务：

- 增加“导出 PDF”入口。
- 生成打印友好的导出预览页。
- 按 `pageIndex / paragraphOrder / sentenceIndex` 排列笔记。
- 输出文档名、导出时间、笔记数量。
- 输出原句、翻译、标签、个人笔记、来源位置。
- 使用浏览器打印保存为 PDF。

预期效果：

- 用户可以把当前文档学习笔记导出为 PDF。
- PDF 内容清晰、可打印、可复习。
- 不导出完整原文页面，降低版权风险。

### 12.7 小阶段七：回归测试与验收

目标：

- 确保第三阶段不破坏第二阶段已有阅读体验。

任务：

- 增加数据 store 的单元或类型测试。
- 扩展桌面端 Playwright 用例：创建笔记、编辑笔记、隐藏边注、拖拽宽度、导出预览。
- 扩展移动端用例：边注默认隐藏，句子笔记可通过底部抽屉查看。
- 回归现有导入、阅读、翻译、高亮、标签、导出 Markdown/TXT 功能。

预期效果：

- 第三阶段核心路径可自动验证。
- 桌面和移动端基础体验都不回退。

## 13. 第三阶段验收标准

| 模块 | 验收标准 |
|---|---|
| 阅读布局 | 桌面端形成原文列和同步边注列，布局稳定 |
| 笔记同步 | 笔记显示在对应句子右侧，滚动时不脱离原句 |
| 笔记创建 | 可以从选中句子创建笔记，并避免重复创建 |
| 笔记编辑 | 可以编辑翻译、标签和个人笔记 |
| 笔记删除 | 删除笔记后边注消失，相关高亮按规则处理 |
| 标签复用 | 笔记标签复用现有 TagRecord，不重复造标签系统 |
| 隐藏展开 | 边注列可以隐藏和恢复 |
| 拖拽宽度 | 边注列宽度可以拖拽调整并持久化 |
| 本地保存 | 刷新页面后笔记和设置不丢失 |
| 小屏适配 | 小屏幕默认隐藏边注，可通过抽屉查看笔记 |
| PDF 导出 | 可以导出当前文档全部学习笔记 |
| 版权边界 | 不导出完整原书页面或完整原书内容 |

## 14. 风险与注意事项

### 14.1 句子切分不准确

PDF 解析后可能出现换行、断句、缩写误判等问题。

应对方式：

- 第一版继续复用现有 `splitSentences`。
- 笔记绑定到现有 `sentenceId` 和 `textHash`。
- 如果后续 parser 版本变化，需要考虑旧笔记迁移或失配提示。

### 14.2 长笔记影响对齐

用户写很长笔记时，右侧卡片可能撑高当前行。

应对方式：

- 卡片默认显示摘要。
- 点击后进入展开编辑态。
- 设置默认最大高度，内部滚动或展开查看。

### 14.3 高亮、标签、笔记删除关系复杂

笔记依赖句子高亮，标签又依赖高亮。删除笔记时如果直接删除高亮，可能误删用户单独创建的标签关系。

应对方式：

- `NoteRecord.highlightId` 明确记录关联高亮。
- 删除笔记前判断该高亮是否仅由笔记创建和使用。
- 如果高亮已有标签或用户显式保留，应只删除笔记，不强制删除高亮。

### 14.4 小屏幕空间不足

同步边注更适合桌面端宽屏阅读。

应对方式：

- 小屏幕默认隐藏边注列。
- 提供底部抽屉式笔记视图。
- 移动端完整边注体验放到后续阶段。

### 14.5 中文 PDF 导出乱码

前端直接生成 PDF 时中文字体容易出问题。

应对方式：

- 第一版使用 HTML 打印保存 PDF。
- 导出预览页使用系统中文字体栈。
- 后续如需一键生成 PDF，再引入稳定字体嵌入方案。

### 14.6 现有导出文本存在编码风险

当前导出相关代码中需要注意中文字符串编码和显示。第三阶段实现 PDF 导出时，应统一检查导出标题、页码、段落文字等中文文案，避免出现乱码。
